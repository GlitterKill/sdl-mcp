# SDLBench 10x Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make SDLBench honestly expose SDL-MCP's real wins/losses so it (a) backs published claims and (b) produces actionable SDL-improvement signals.

**Architecture:** Three phases. P0 fixes the **truth gap** created by mixing fixture-mode tautologies with real behavior counts (no new harness surface, ~3 files). P1 builds the **proof surface** — repo scaling, attribution, amortization, ground-truth coverage (new harness modules, extends SessionRecord). P2 adds **rigor** — multi-turn tasks, stats, fairness audit, multi-agent, claim gates.

**Tech Stack:** Node.js ESM (`.mjs`), `node:test`, tiktoken subprocess, SDL-MCP HTTP `/api/observability/*` polling, Codex session JSONL parsing. No new deps.

**User decisions (locked):**
- Save path: `sdlbench/docs/plans/2026-06-26-sdlbench-10x.md` (co-located with harness).
- Delivery: P0 lands first as a standalone deliverable; regroup before P1/P2.

**Grounding (cited findings):**
- Real behavior shows SDL **3.5x tokens / 3.4x cost** on the same 4 tasks (`sdlbench/results/real-behavior-*.jsonl`), but the fixture-mode summary advertises **+20.4% savings** (`summary.json`).
- `countSessionTokens` (`sdlbench.mjs:966`) sets `saved = rawEquivalent - total` from hand-written `context.raw` — a tautology in fixture mode.
- `analyzeSessions` (`sdlbench.mjs:244`) mixes fixture + behavior records into one sum.
- `buildChartModel` (`viewer/app.mjs:9`) sums `tokens.saved` across modes.
- `repos.lock.json:5-7` already lists moshi / rust-analyzer / kubernetes but `sdlbench/tasks/matrix.json:3` only loads `fixture.tasks.json`.
- `sdlbench.mjs:174` hardcodes `workflow.toolCalls=2`/`fileReads=1` instead of parsing them.
- `pricing.json` has no cached-input rate even though behavior records show `cachedInput` ~95%.
- SDL HTTP server exposes `/api/observability/{snapshot,timeseries,beam-explain,stream}` (Bearer-gated) — `src/cli/transport/http.ts:928-1008`.
- Existing `findCodexSessionTokenCounts` (`sdlbench.mjs:1021`) only parses `session_meta` + `event_msg(token_count)`; the JSONL also has `function_call` items used for attribution.

- Whether `claude.mjs` ships functional or stubbed in P2.4 depends on the Claude Code transcript schema availability — fetch the schema via context7 in step 1 of P2.4 and pick the most authoritative format; ask the user if the schema is not documented.
