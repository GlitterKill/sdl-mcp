# native/ - Rust Native Addon (napi-rs)

## OVERVIEW
Optional Rust addon for performance-critical paths: parallel file parsing, community detection (Louvain), and process tracing. Falls back to TS implementations when unavailable.

## STRUCTURE
- `Cargo.toml` - Rust crate config (cdylib via napi-rs)
- `index.js` - Platform-detection loader for prebuilt `.node` binaries
- `sdl-mcp-native.node` - Prebuilt binary (win32-x64)
- `src/` - Rust source
  - `extract/` - File parsing + symbol extraction (11 language grammars)
  - `cluster/` - Louvain community detection (petgraph + rayon)
  - `process/` - Call-chain tracing
  - `parse/` - Tree-sitter AST parsing
  - `lang/` - Language-specific handlers
- `npm/` - Per-platform prebuilt packages (6 targets)

## BUILD

```bash
npm run build:native          # Release build
npm run build:native:debug    # Debug build
```

Requires: Rust toolchain + MSVC on Windows

## PLATFORMS (prebuilt)
darwin-arm64, darwin-x64, linux-arm64-gnu, linux-x64-gnu, linux-x64-musl, win32-x64-msvc

## RUST DEPS
rayon (parallelism), petgraph (graph algorithms), tree-sitter + 11 grammars, sha2/hex (fingerprinting), ignore (gitignore scanning)

## FALLBACK PATTERN
TS code checks `SDL_MCP_DISABLE_NATIVE_ADDON` env. If set or addon missing:
- Parsing: `src/indexer/workerPool.ts` (Node.js worker threads)
- Clustering: `src/graph/cluster.ts` (Label Propagation)
- Processes: `src/graph/process.ts` (DFS tracer)

## CONVENTIONS
- Tests always run with `SDL_MCP_DISABLE_NATIVE_ADDON=1`
- Parity tests: `npm run test:native-parity` (compares Rust vs TS output)
- TypeScript bindings auto-generated by napi-rs
