The nightly benchmark matrix on 2026-03-26 failed because scripts/real-world-benchmark.ts still treated buildSlice() as returning a raw GraphSlice after src/graph/slice.ts changed it to return { slice, retrievalEvidence, hybridSearchItems }. The first my-repo run then crashed at slice.cards.length, and later repo-scoped matrix runs reopened the same Ladybug path in a bad WAL state because the benchmark script exited without awaiting closeLadybugDb(). A small wrapper helper plus top-level finally cleanup fixes the primary API drift and prevents one benchmark failure from poisoning later matrix entries for the same repo.
