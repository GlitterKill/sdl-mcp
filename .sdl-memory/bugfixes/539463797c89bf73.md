Two card quality fixes applied (2026-03-22):

1. Summary quality: `generateSummary` in src/indexer/summaries.ts now returns `string | null`. When no JSDoc exists, returns null for classes/interfaces/types/variables. For functions, builds type-based summaries from extractSimpleType(param.type) and returns type — returns null when no type info is available. New `isNameOnlySummary` detects stale name-only summaries and clears them during incremental reindexing (self-healing in both process-file.ts and rust-process-file.ts). Rust native/src/extract/summary.rs mirrors changes.

2. Canonical test mapping: BFS in src/graph/metrics.ts now uses edgeCost(edgeType): call=1, config=1.5, import=2. MAX_BFS_DEPTH renamed to MAX_BFS_COST=12. batchComputeCanonicalTests uses relaxed visited-set (update-if-better). Incremental indexing now writes canonical test updates for ALL symbols via new upsertCanonicalTestBatch in ladybug-metrics.ts, fixing stale data for unchanged symbols.

Key detail: isNameOnlySummary filters noise words BEFORE deconjugate stemming to avoid artifacts like 'alias'->'alia'. The deconjugate function handles sibilant endings (sses/zes/xes/shes/ches strip -es, others strip -s).
