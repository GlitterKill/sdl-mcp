After the repo raised its minimum engine to Node 24, the CI test matrix still ran Node 22 jobs and failed the doctor capability suite because `doctorCommand` correctly exits on unsupported Node versions. Separately, `tests/unit/doctor-confidence-capabilities.test.ts` raced under the default node:test concurrency because both subtests mutate global `process.exit`, `console.log`, and the shared Ladybug connection; adding `{ concurrency: 1 }` makes the suite deterministic. The workflow also no longer forces `npm rebuild tree-sitter ...`; instead it verifies the installed tree-sitter packages load, which avoids Node 24 native rebuild failures while still checking parser availability.
