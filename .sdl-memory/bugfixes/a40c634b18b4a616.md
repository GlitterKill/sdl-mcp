In src/code/windows.ts, the countBracesOutsideStrings function counted `{` and `}` inside template literal interpolation expressions (e.g., `${obj.method({ key })}`) as real code braces. This caused expandToBlock to compute wrong block boundaries for files with nested template literals. Fix: added `else` clauses so open++ and close++ only execute when interpDepth === 0 (not inside interpolation). The `{` that opens a nested object inside interpolation now only increments interpDepth, and `}` that closes it only decrements interpDepth, without affecting the brace count used for block expansion.
