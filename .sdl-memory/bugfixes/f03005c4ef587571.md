Every log line (when file logging enabled) called `appendFileSync`, a blocking open-write-close syscall. Under debug logging in hot paths (beam search, indexing), this produced thousands of blocking I/O calls per operation.

Fix: Replaced with a lazily-initialized `createWriteStream(logFilePath, { flags: 'a' })`. The stream buffers writes and flushes asynchronously. Log rotation closes the stream (`logStream.end()`), renames the file, sets stream to null (re-created on next write). Added `closeLogStream()` export for shutdown cleanup. The `flushLogger()` function now ends the stream to flush pending writes.

Key detail: `rotateLogFile` still uses sync operations (statSync, renameSync) since rotation is infrequent and needs to be atomic relative to writes — just closes the stream first.
