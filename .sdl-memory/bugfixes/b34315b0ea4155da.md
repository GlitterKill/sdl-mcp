Two multi-statement delete operations were not wrapped in transactions:

1. `deleteMemoryEdges` (ladybug-memory.ts): Three separate `exec()` calls delete HAS_MEMORY, MEMORY_OF, and MEMORY_OF_FILE edges. If the second or third fails, memory node has partially deleted edges.

2. `deleteEdgesByFileId` (ladybug-edges.ts): Read symbolIds + delete outgoing + delete incoming edges. Partial failure leaves dangling reverse references.

Fix: Both wrapped in `withTransaction(conn, async (txConn) => { ... })` with all inner queries using `txConn`. Note: `withWriteConn` (used by callers) provides write serialization but NOT transactional atomicity — always use `withTransaction` for multi-statement atomic operations.
