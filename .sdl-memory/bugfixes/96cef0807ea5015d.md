resolveExecutable() and getVersion() in src/runtime/runtimes.ts used execSync with template literal interpolation (`where ${name}`, `${executable} ${versionFlag}`), which spawns a shell. While the name values come from a hardcoded RUNTIME_TABLE, this was a defense-in-depth violation. Fixed by switching to execFileSync with array-form arguments: execFileSync("where", [name]) / execFileSync(executable, [versionFlag]). Also fixed getVersion being called with bare candidate name instead of resolved absolute path from resolveExecutable().
