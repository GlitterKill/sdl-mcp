The `test:stress` package script could not run on Node 24 by executing `tests/stress/run-stress.ts` directly because the stress TS sources use `.js` import specifiers and the raw source tree has no sibling `.js` files. The fix was to compile `tests/stress` to `dist/tests/stress`, run the compiled entrypoint with `--skip-build`, and replace hard-coded relative `../../../dist/...` or source-relative root assumptions with runtime helpers that detect the real `dist/` and repo root from either source or compiled layouts. This also avoids a Windows `EPERM` during `clean-dist` by preventing the compiled orchestrator from rebuilding `dist` a second time.
