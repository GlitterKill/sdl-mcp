Code review round 8 (2026-03-25) found 7 setTimeout/setInterval calls missing .unref(), which prevents clean Node.js process exit during shutdown.

Fixed files:
1. src/delta/blastRadius.ts:410 — diagnostics timeout
2. src/graph/slice/beam-search-engine.ts:1431 — parallel scorer timeout
3. src/indexer/watcher.ts:291 — reindex retry delay (.unref() chained)
4. src/indexer/watcher.ts:316-320 — debounce timer (restructured to assign then .unref())
5. src/live-index/debounce.ts:60 — debounce job scheduler timer
6. src/live-index/reconcile-worker.ts:88 — waitForIdle timeout
7. src/runtime/executor.ts:207 — child process execution timeout

Pattern: Any setTimeout/setInterval in long-lived server code MUST call .unref() to avoid keeping the process alive after shutdown cleanup. Exception: timers that intentionally prevent exit (e.g., force-exit timeout in shutdown.ts).

The codebase already had .unref() on many timers (concurrency.ts, idle-monitor.ts, session-manager.ts, main.ts cleanup interval, executor.ts kill timeout, db drain timeout). These 7 were missed in earlier rounds.
