The local dev environment has 171 pre-existing type errors from three sources that do NOT prevent JS emission (noEmitOnError is not set):

1. **tree-sitter type defs** (~140 errors): `tree-sitter.d.ts is not a module` + cascading `Parameter 'c' implicitly has 'any' type`. Affects all adapter files, skeleton.ts, hotpath.ts, extractors. The tree-sitter npm package types don't resolve on this system/Node version.

2. **globSync/glob** (~3 errors): `Module "node:fs" has no exported member 'globSync'` in metrics.ts and tsParser.ts; `Module "node:fs/promises" has no exported member 'glob'` in fileScanner.ts and import-resolution adapters. These are Node 22+ APIs that TypeScript's type definitions don't know about.

3. **OpenTelemetry** (1 error): `@opentelemetry/resources` doesn't export `resourceFromAttributes` in the installed version. This also causes runtime test failures — tests that transitively import `dist/util/tracing.js` crash with SyntaxError.

CI passes because it uses ubuntu with properly installed native modules. The build still emits 305 JS files despite errors.
