The HTTP transport logs the auto-generated auth token in plaintext to stderr (src/cli/transport/http.ts:1262). This is expected behavior, not a security issue. The token must be visible so that local processes (like `sdl-mcp index` delegating to a running server, or the dev UI) can authenticate. stderr is the correct channel since stdout is reserved for JSON-RPC in MCP servers. The token is also written to the PID file for process discovery.
