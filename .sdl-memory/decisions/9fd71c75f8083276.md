Multiple module-level Maps grew without bound over long-running server sessions. Established a consistent eviction pattern across all caches:

1. `graphSnapshotCache.ts`: MAX_CACHED_REPOS=3, evicts oldest by `createdAt` timestamp
2. `prefetch.ts`: MAX_PREFETCH_ENTRIES_PER_REPO=500, evicts entries older than 5 minutes
3. `overview.ts`: MAX_OVERVIEW_CACHE_ENTRIES=20, evicts oldest by `expiresAt`
4. `metrics.ts`: MAX_METRICS_CACHED_REPOS=5 for both churnCacheByRepo and testRefCacheByRepo, evicts first Map entry (oldest by insertion order)

Pattern: After `map.set()`, check `map.size > MAX`, iterate to find oldest, delete. Simple and predictable. No external LRU library needed since Maps preserve insertion order and cache sizes are small.
